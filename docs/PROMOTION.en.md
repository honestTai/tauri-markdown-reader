## Short Copy

If your project knowledge lives in scattered Markdown files but you do not want to open an IDE every time you need to read a document, Markdown Reader turns a local folder into a desktop document library.

It supports full-text search, outline navigation, reading position restore, favorites, pinned documents, light editing, local image insertion, and export to Word, PDF, plain text, and reading HTML. Files stay local by default, rendered Markdown is sanitized, and desktop file operations are scoped to opened Markdown workspaces.

It is a good fit for project README files, PRDs, technical plans, meeting notes, troubleshooting records, and personal knowledge bases.

It is not trying to replace Typora, Obsidian, or VS Code. The scope is narrower: when you only need to open a project folder, search, read, favorite, and export the Markdown documents inside, you should not have to enter a full development environment.

## Community Post

I built Markdown Reader, a local-first Markdown document library for project docs.

It is not another Markdown editor. It is a local workbench for reading and searching PRDs, README files, technical plans, meeting notes, and troubleshooting logs. Open a folder, search across files, jump through outlines, restore reading positions, favorite or pin important docs, make light edits, and export to Word/PDF/HTML.

Built with Tauri + React + TypeScript. Windows and macOS builds are available.

GitHub: https://github.com/honestTai/tauri-markdown-reader

## Highlights

- Local-first: open a folder or a single `.md` file without relying on the cloud.
- Document-library view: organize Markdown by folder, favorites, pinned documents, recent files, and update time.
- Full-text search: search filenames, titles, frontmatter, body text, and snippets.
- Reading experience: three-column workbench, right-side outline, word count, image status checks, and reading position restore.
- Light editing: save-before backup, local image insertion, and source editing.
- Multiple exports: Word, PDF, Markdown, plain text, and reading HTML.
- Security hardening: DOMPurify sanitization, strict Tauri CSP, workspace-scoped file access, and automated CI checks.
- Updated docs: README now supports Chinese / English switching for both local and international audiences.

## Positioning

- Not a writing-first editor: the focus is reading, searching, and reusing existing project docs.
- Not a cloud note app: local-first by default, no sync account required.
- Not an IDE plugin: useful when you want to read docs without entering development context.
- Not a large knowledge-base system: open a folder and use it directly.
